# 🌍 Disaster Management Database

This project implements a relational database system (DisasterDB) for managing disaster events, impacts, and response operations.

---

## 📊 Project Overview

The system models the full lifecycle of a disaster:

* Event information (type, location, time)
* Impact analysis (deaths, injuries, economic loss)
* Organizations and response actions
* Aid and operational activities

---

## 🛠️ Technologies Used

* Microsoft SQL Server (T-SQL)
* Relational Database Design
* Constraints & Triggers

---

## 🧩 Features

* Complex relational schema with multiple entities
* Foreign keys and data integrity constraints
* Triggers for validation and automation
* Real-world disaster data modeling

---

## 📐 Database Design

* EER Diagram included
* UML Diagram included

---

## 📌 Key Concepts

* Event-based modeling
* Natural keys usage
* Data integrity enforcement
* Query-based analysis

---

## 🚀 Purpose

Designed to support querying and analysis of disaster data, including:

* Impact reporting
* Organization response tracking
* Aid distribution analysis
